### 🛠️ Ubuntu $\times$ AI開発：最強コマンド集

#### 1. システム管理・環境設定（OSの基本）
| コマンド | 意味・役割 | 使いどころ |
| :--- | :--- | :--- |
| `sudo apt update` | パッケージリストの更新 | ソフトを入れる前に必ず行う |
| `sudo apt install -y [ソフト名]` | ソフトのインストール | 新しいツール（ghなど）を入れる時 |
| `export LANG=ja_JP.UTF-8` | 一時的な日本語化（環境変数） | `ls` などで日本語を正しく表示させたい時 |
| `ls` | ファイル・フォルダの一覧表示 | 今どこに何があるか確認する時 |
| `cd [フォルダ名]` | フォルダ（ディレクトリ）の移動 | 作業したいフォルダへ移動する時 |

#### 2. ファイル操作とAI連携（パイプとリダイレクト）
| コマンド | 意味・役割 | 使いどころ |
| :--- | :--- | :--- |
| `cat [ファイル名]` | ファイルの中身を画面に出す | 中身を確認したり、AIに渡したりする時 |
| `\|` (パイプ) | 前の出力を次のコマンドへ渡す | `cat` した内容を `ollama` に流し込む時 |
| `> [ファイル名]` | **上書き**保存（中身を入れ替える） | AIの回答を新しいverとして保存する時 |
| `>> [ファイル名]` | **追記**保存（後ろに付け足す） | AIの回答をログとして蓄積したい時 |
| `\| tee [ファイル名]` | **画面表示 ＋ 上書き**保存 | 結果を読みながらファイルにも保存したい時 |
| `\| tee -a [ファイル名]` | **画面表示 ＋ 追記**保存 | 結果を読みながらログに付け足したい時 |
| `cat prompt.txt code.py \| ollama ...` | 指示書とコードをまとめてAIへ | **【ハイブリッド運用】** 指示とデータを一気に渡す時 |

#### 3. バージョン管理（Git $\times$ GitHub CLI）
| コマンド | 意味・役割 | 使いどころ |
| :--- | :--- | :--- |
| `gh auth login` | GitHubへのログイン認証 | 最初に一度だけ、PCとGitHubを繋ぐ時 |
| `gh repo create [名] --public ...` | GitHubに公開リポジトリを作成 | 新しいプロジェクトをネットに公開する時 |
| `gh repo view --web` | ブラウザでリポジトリを開く | アップロード後の内容を画面で確認したい時 |
| `git init` | Git管理の開始（初期化） | フォルダをGit管理下に置く最初の操作 |
| `git add .` | 全ファイルを「荷造り」する | 更新したファイルを保存対象に選ぶ時 |
| `git commit -m "メッセージ"` | 変更を「確定」して保存する | 意味のある区切りで保存ポイントを作る時 |
| `git push origin master` | ネット上のGitHubへ送信する | 保存した内容を世界（ネット）に公開する時 |

---

### 💡 迷った時の「黄金の3連撃」メモ
ファイルを更新してGitHubに上げたい時は、迷わずこの順番で！

1.  **`git add .`** （選ぶ）
2.  **`git commit -m "〇〇を修正"`** （確定させる）
3.  **`git push origin master`** （送る）

### 💡 AIへの「最強の指示」テンプレート（保存用）
指示書ファイル（`prompt.txt`）に書いておくべき必須ルール：
*   **バッククォート（\`\`\`）禁止**
*   **解説はすべて `#`（コメント）で記述**
*   **コード部分はそのまま実行可能な形式で**
*   **修正なしに `python3` で実行できる状態で出力**
